_취준생들이 취업 준비를 위한 면접을 체험하고 싶으신가요?\
취준을 위한 면접 준비가 필요한데 적절한 상대가 없으신가요?\
면접을 준비하기 위한, 필요한 면접매칭을 도와주고\

ai로 이력서 기반 질문생성으로 면접관들에게 질문에 대한 도움을주고\

### 협업과정

> 1. 기본적으로 디스코드 + 노션을 활용하였고,디자이너와 협업을 위한 피그마까지  활용하였다. 디스코드에선 채널 분리를 하여
> 실시간 소통을하였다.(직군별 페이지,error 공유 페이지, 진행상황 공유 페이지 등) 노션에서는 문서화가 필요한 소통을 하였다.
> (api, error 정리 ) 피그마에선 디자인을 해놓으면 회의때 궁금한것이나 필요한것들을 피그마에 직접 멘트를 달고 , 답장을 
> 하는식으로 진행하였고 프론트, 백엔드 요구사항등은 디스코드에 남겼다.
> 2. 프론트에선 백엔드를위해서 폴더구조에 대한 간단한 설명을 README로 작성하여 프론트를 작성할때 참고 할수있도록 하였다.

## 기술스택

> 1. 백엔드와 의 협업을 위해 + csr 정도면 충분한거같아서 React+vite+typescript를 채택하였다.
> 2. 프로젝트 특성상 node.js를 사용하는게 맞고(실시간 서비스), 풀스택으로 할줄알지만, 
> 백엔드 팀원들이 java 를 사용해서 java spring boot 프레임워크 를 사용하였다.
> 3. 리액트 쿼리 훅을 사용하여 client와 server를 분리하여 관리하였다.
> 4. rhf - 리액트 훅 폼 을 사용하여, 로그인 로그아웃 등 데이터를 입력하는 post부분에 대해서 간편하게 관리
> 해주는 툴이다. 리액트 훅 폼을 훅으로 만들었고 form과 관련된 ui는 상태 관리 툴중에 하나인 context api로 관리한다.
> 나머지는 redux 를 통해 관리
> 5. ui 관련해선 radxi/ui 를통해 틀을잡았고, tailwindcss 로 전체적으로 스타일링을 했다.
> 6. vercel 을 활용하여 serverless로 구성 open ai를 자바를 사용하지않고 구성하였다. open ai를 통해 번역기를 만들었다.

## 주요기능

### 매칭 기능

> 1. 매칭 할때 필요한 정보들을 입력한다. 이정보는 매칭 + 채팅에서만 활용이 된다. 매칭은 아주 단순하게 진행된다.
> 2. 3분뒤에는 조건이 완화되기는하나 모든 조건이 똑같은 경우에만 매칭을 잡아주는 형태로 가게된다.
> 3. 이력서가 등록 되어있어야하면 등록되어있지 않다면 마이페이지 > 이력서 페이지에서 등록해야한다.
> 여기 등록된 정보가 연동되는 형태이다.
> 4. 매칭정보는 모두 들여다볼수있도록 입력을때 다음 다음 넘어가도록 하였다.


### 비디오 챗

> 1. 채팅에서 모두가 동의하면 비디오 챗으로 넘어오는 상대 화면을 클릭하면 이력서 및 질문이 보인다.(ai생성)
> 2. 비디오 챗에서 ai질문을통해 질문을 할수있으면 추가적인 질문을 등록하고 질문할수있다.
> 3. 질문에 대한 답을 듣고 거기에 피드백을 작성할 수 있다. 피드백 생성시간이 존재해서 시간이필요하다.
> 4. 각각 질문에대한 피드백은 마이페이지 > 피드백에 저장이 되며 ai가 간단히 정리해서 요약 해준다.
> 상세 피드백도 존재.


### 채팅 기능

> 1. 매칭 다음에 채팅칸으로 넘어오는데 채팅에선 서로에대한 간단한 이력 확인 및 대화를 할수 있다.
> 2. 간단한 대화후 동의한다면 비디오챗으로 넘어갈수있는데, 동의하기 버튼을 통해 넘어갈수 있으며,
> 모두가 동의를 해야만 넘어갈수 있다. 맘에 들지않으면 취소 가능하다.

## 프로젝트에서 문제점 해결한 과정

> 1. chatgpt 이후로는 사실 거의 모든 문제를 chatgpt로 해결한다. 그런데 모든 문제가 chatgpt로 해결되지는 않는다.
> 만약 최신으로 변경된경우라면 따로 언급하지 않으면 옛날 데이터를 그대로 주는경우가많았다 
> => 해결이 잘되지 않는경우, 공식문서를 따라가 최신버전을 확인, 변경점을 파악하고, 버전을 그대로 다시물어본다.
> 2. react query로 로그인 변화 에 따른 header의 변화를 주고싶엇는데 react query auth + 비로그인 강제이동 사용
> 중이였는데 2개가 충돌나면서 문제가 생겨서 질문을 했으나 계속 해결을 못해주고 똑같은 답변만 했다.
> => 이런경우 직접 주제를 검색해서 다른 코드에서 아이디어를 얻은 경우가있었다.
